# Security Policy

## Reporting a Vulnerability

If you believe you found a security issue in SDDP, please report it privately.

1. Preferred: open a GitHub Private Security Advisory in this repository.
2. Fallback: use the maintainer security contact published on the repository profile or release notes.

## What to Include

- Affected version or commit
- Reproduction steps
- Impact assessment
- Suggested fix, if available

## Notes

- Security reports are handled on a best-effort basis.
- Fixes are prepared in the internal source repository first and then republished to the public repository.
- Please do not open public issues for unpatched vulnerabilities.
